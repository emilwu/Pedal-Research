# MVP 範圍定義

## MVP 原則

- 聚焦核心功能，快速驗證市場需求
- 保留架構彈性，便於後續擴展
- 先做最小可用，再根據反饋迭代

## 功能優先級總覽

| 優先級 | 功能 | 類型 | MVP 狀態 |
|-------|------|------|---------|
| P1 | 設備研究請求 | AI 驅動 | ✓ 包含 |
| P2 | 設備數據庫瀏覽 | 靜態 | ✓ 包含 |
| P3 | 智慧配對 | AI 驅動 | ✓ 包含 |
| P4 | 訊號鏈生成 | AI 驅動 | ✗ 後續版本 |

## MVP 功能清單

### 1. 用戶系統（基礎）

| 功能 | MVP | 說明 |
|-----|-----|------|
| 註冊 | ✓ | Email 註冊 |
| 登入/登出 | ✓ | |
| 密碼重設 | ✓ | Email 驗證 |
| 個人資料 | ✓ | 基本資料編輯 |
| 使用額度查看 | ✓ | 顯示今日剩餘配對次數 |
| 訂閱管理 | ✗ | 後續版本 |
| 第三方登入 | ✗ | 後續版本 |

### 2. 設備數據庫

| 功能 | MVP | 說明 |
|-----|-----|------|
| 設備列表 | ✓ | 分類顯示（效果器/吉他/音箱） |
| 搜尋 | ✓ | 關鍵字搜尋 |
| 篩選 | ✓ | 依類型、品牌篩選 |
| 設備詳情頁 | ✓ | 完整規格、設定建議 |
| 初始數據 | ✓ | 導入現有 27+ 設備資料 |
| 訪問權限 | 公開 | 無需登入即可瀏覽 |

**支援設備類型：**
- 效果器 (Pedals)
- 吉他 (Guitars)
- 音箱 (Amps)

### 3. 設備研究請求

| 功能 | MVP | 說明 |
|-----|-----|------|
| 提交研究請求 | ✓ | 輸入品牌+型號 |
| 排隊系統 | ✓ | FIFO 排隊 |
| 預估等待時間 | ✓ | 顯示預估完成時間 |
| 背景 AI 處理 | ✓ | 非同步處理 |
| Email 通知 | ✓ | 完成時發送 Email |
| 結果歸屬 | 公開 | 自動進入公開數據庫 |
| 單次付費插隊 | ✗ | 後續版本 |
| 訂閱用戶優先 | ✗ | 後續版本 |

**MVP 限制：**
- 免費無限次請求（需登入）
- 所有用戶共用處理資源，依序排隊

| 查看清單 | ✓ | 分類顯示 |

**需登入才能使用**

### 5. 智慧配對

| 功能 | MVP | 說明 |
|-----|-----|------|
| 選擇配對模式 | ✓ | 個人清單 / 假設性選擇 |
| 選擇吉他 | ✓ | 從清單或數據庫選擇 |
| 選擇音箱 | ✓ | 從清單或數據庫選擇 |
| 選擇音樂風格 | ✓ | 預設選項 + 自定義修改 |
| AI 配對計算 | ✓ | 即時計算 |
| 結果展示 | ✓ | 推薦設備 + 設定建議 |
| 每日限額 | ✓ | 免費用戶每日有限次數 |

**音樂風格選項（預設）：**
- Jazz
- Neo Soul
- Funk
- Post Rock
- Fusion
- Blues
- 自定義（用戶手動輸入）

**MVP 限額：**
- 免費用戶：每日 N 次（具體數字待定）

|-----|-----|------|

## MVP 不包含的功能

以下功能明確排除在 MVP 範圍外：

| 功能 | 原因 | 預計版本 |
|-----|------|---------|
| 訊號鏈生成 | 複雜度高，先驗證基礎功能 | v2 |
| 設備替代性分析 | 依賴訊號鏈功能 | v2 |
| 付費訂閱系統 | 先驗證產品價值 | v2 |
| 私人數據庫 | 需訂閱系統支援 | v2 |
| 社群功能 | 非核心功能 | v3+ |
| 英文介面 | 先聚焦單一語言 | v2 |
